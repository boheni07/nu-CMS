## 1. 시스템 개요
nu-CMS는 전자정부 표준 프레임워크 기반의 안전하고 확장 가능한 콘텐츠 관리 시스템입니다.
반응형 웹 디자인을 지원하며, 강력한 보안 정책(RBAC, 암호화)과 사용자 친화적인 UI(React, Ant Design)를 제공합니다.

---

## 2. 로그인 및 대시보드

### 로그인
- **접속 주소**: `/login`
- **계정**: 관리자가 발급한 ID/PW 사용
- **특이사항**: 5회 이상 로그인 실패 시 계정이 잠금 처리됩니다.

### 대시보드
로그인 후 처음 보이는 화면으로, 시스템의 전반적인 상태를 한눈에 확인할 수 있습니다.
- **방문자 통계**: 최근 7일간의 페이지뷰(PV) 추이 그래프
- **콘텐츠 현황**: 전체 게시물 수, 승인 대기 건수 등
- **시스템 알림**: 최근 에러 로그나 보안 경고 메시지

## 3. 콘텐츠 관리
사이트의 일반 페이지(About Us, 회사소개 등) 내용을 작성하고 수정하는 메뉴입니다.

- **경로**: CMS > 콘텐츠 관리 > 콘텐츠 목록
- **주요 기능**:
    1. **목록 조회**: 제목, 상태, 등록일 기준 검색 가능
    2. **신규 등록**: 우측 상단 '신규 등록' 버튼 클릭
    3. **에디터**: WYSIWYG 에디터를 통해 텍스트, 이미지 서식 적용 가능
    4. **버전 관리**: 수정 시마다 버전이 기록되며, 이전 버전으로 원클릭 복구 가능

### 카테고리 관리
콘텐츠를 체계적으로 분류하기 위한 카테고리를 관리합니다.
- **경로**: CMS > 사이트 관리 > 카테고리 관리
- **기능**:
    - **목록 조회**: 트리(Tree) 구조로 카테고리 계층 확인
    - **신규 등록**: 하위 카테고리 추가 및 최상위 카테고리 생성
    - **수정 및 삭제**: 카테고리명 변경, 사용 여부 설정, 삭제 처리

## 4. 메뉴 및 레이아웃 관리
사이트의 내비게이션 구조와 화면 구성을 제어합니다.

### 메뉴 관리
- **경로**: CMS > 사이트 관리 > 메뉴 관리
- **사용법**:
    - 좌측 트리에서 메뉴 선택 또는 드래그 앤 드롭으로 순서 변경
    - 우측 패널에서 메뉴명, 연결 URL, 사용 여부 설정 후 '저장'

### 레이아웃 및 위젯
- **템플릿 설정**: 헤더, 푸터, 사이드바 영역별 위젯 배치
- **위젯 관리**: 배너, 팝업, 퀵메뉴 등을 생성하고 관리

## 5. 게시판 관리
공지사항, 자료실, Q&A 등 다양한 형태의 게시판을 생성하고 관리합니다.

### 게시판 마스터 (생성/설정)
- **경로**: CMS > 부가 서비스 > 게시판 관리
- **기능**:
    - **게시판 생성**: ID, 이름, 유형(일반/공지), 답글/첨부파일 허용 여부 설정
    - **관리**: 생성된 게시판의 '관리' 버튼 클릭 시 해당 게시판의 글 목록으로 이동
    - **삭제**: 게시판 및 하위 게시물 전체 삭제 (주의 필요)

### 게시물 관리 (글쓰기)
- **경로**: 게시판 마스터 목록 > '관리' 버튼 클릭
- **기능**:
    - **목록 조회**: 게시글 목록 확인 (제목 클릭 시 **미리보기 모달** 팝업)
    - **미리보기**: 게시물 상세 내용을 페이지 이동 없이 모달창에서 즉시 확인 가능 (ESC 키로 닫기)
    - **신규 등록**: 우측 상단 버튼을 통해 글 작성 (전용 웹 에디터 사용)
    - **수정 및 삭제**: 본인 글 또는 관리자 권한으로 수정/삭제 가능

## 5. 사용자 및 권한 관리

CMS의 권한 체계는 **회원(Member) - 역할(Role) - 메뉴(Menu)**의 3단계로 연결됩니다.

### 5.1 권한 설정 흐름
1.  **역할 정의**: [역할 관리] 메뉴에서 새로운 역할을 생성합니다. (예: `게시판 관리자`)
2.  **메뉴 권한 부여**: 생성된 역할을 선택하고, 우측 '메뉴 접근 권한 설정' 트리에서 접근 가능한 메뉴를 체크하여 저장합니다.
3.  **회원에게 역할 할당**: [회원 관리] 메뉴로 이동하여, 특정 회원의 **[권한]** 버튼을 클릭합니다.
4.  **역할 매핑**: 해당 회원에게 부여할 역할(복수 선택 가능)을 선택하고 저장합니다.

> **참고**: 회원은 여러 역할을 가질 수 있으며, 부여된 모든 역할의 권한이 합집합(Union)으로 적용됩니다.

### 5.2 회원 관리
시스템 접근 권한을 가진 사용자를 관리합니다. (관리자 전용)

- **경로**: CMS > 시스템 관리 > 회원 관리
- **기능**:
    - **신규 회원 등록**: [신규 회원] 버튼을 통해 ID, 이름, 비밀번호, 이메일 등 입력 (비밀번호 암호화 저장)
    - **정보 수정**: [수정] 버튼 클릭 시 회원 정보 수정 (비밀번호 변경 가능)
    - **상태 관리**: [잠금] / [복구] 버튼으로 회원 상태 즉시 변경
    - **권한 확인**: 목록에서 회원이 보유한 **역할(Role)**을 태그 형태로 확인 가능

### 역할(Role) 관리
- **경로**: CMS > 시스템 관리 > 역할 관리
- **기능**:
    - 역할 정의 (예: ADMIN, USER, GUEST)
    - **권한 매핑**: 각 역할별로 접근 가능한 메뉴를 체크박스로 설정

## 7. 미디어 라이브러리
이미지, 문서 등 파일을 통합 관리하는 저장소입니다.

- **경로**: CMS > 콘텐츠 관리 > 미디어 라이브러리
- **특징**:
    - **드래그 앤 드롭 업로드** 지원
    - 이미지 업로드 시 썸네일 자동 생성
    - 보안 URL을 통한 파일 접근 (비로그인 사용자 접근 차단)

---

## 8. 보안 및 감사 로그
시스템의 주요 활동 이력을 모니터링합니다.

- **경로**: CMS > 시스템 관리 > 감사 로그
- **기록 항목**:
    - 누가(User), 언제(Time), 어디서(IP), 무엇을(URL/Method) 했는지 기록
    - 데이터 변경(등록/수정/삭제) 작업 자동 추적

---

## 9. 워크플로우 승인 시스템
콘텐츠 게시 전 승인 절차를 거치도록 하는 시스템입니다.

- **경로**: CMS > 승인 관리 > 요청 목록 / 결재함
- **주요 기능**:
    - **승인 요청**: 콘텐츠 작성 후 '승인 요청' 버튼을 통해 관리자에게 게시 허가 요청
    - **결재 처리**: 관리자는 요청된 항목을 검토 후 '승인' 또는 '반려' 처리
    - **이력 관리**: 요청 및 처리 내역이 모두 기록되어 추후 확인 가능

---

## 10. 디지털 자산 관리 (DAM)
이미지, 동영상 등 미디어 파일의 메타데이터를 관리하고 검색을 용이하게 합니다.

- **경로**: CMS > 콘텐츠 관리 > 미디어 라이브러리 (DAM 모드)
- **기능**:
    - **메타데이터 입력**: 파일 업로드 시 저작권자, 만료일, 태그 등 부가 정보 입력
    - **고급 검색**: 태그, 파일 유형, 기간별 필터링 검색 지원
    - **버전 관리**: 동일 파일의 수정 이력 관리
